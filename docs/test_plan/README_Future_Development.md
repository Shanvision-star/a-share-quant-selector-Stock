# 项目扩展功能开发手册

## 项目简介
**仓库名称**: `Shanvision-star/a-share-quant-selector-Stock`  
**项目描述**: 本项目基于开源仓库 [a-share-quant-selector](https://github.com/Dzy-HW-XD/a-share-quant-selector) 进行优化与改进，主要用于量化策略的选择与实现。

**当前语言构成**:
- Python：92.3%（主要后端逻辑与数据处理语言）
- JavaScript：3.7%（前端交互逻辑）
- CSS：2.0%（前端样式）
- HTML：1.8%（网页模板）
- Shell：0.2%（支持脚本）

---

## 后续功能实现概述
目标是将该项目扩展为支持以下功能的多端统一服务：
1. **网页端**：一个交互式网站，用于展示量化策略运行结果、实时图表、数据监控等。
2. **移动端 APP**：实现策略监控、数据查看、运行状态通知等功能，支持 Android 和 iOS。
3. **UI 设计**：统一设计语言与交互逻辑，提供现代化用户界面，兼顾响应式布局与易用性。

## 功能开发计划

### 1. **后端服务**
#### 技术选型
- **框架**: 使用 Flask、Django 或 FastAPI。
- **数据库**: PostgreSQL 作为关系型数据库，SQLite 可用于测试。
- **API 形式**: 提供 Restful API，未来可扩展为 GraphQL。

#### 核心后端模块
1. **用户管理模块**：
   - 用户注册、登录、身份认证（建议使用 JWT）。
2. **量化策略模块**：
   - 提供添加、修改、删除策略的 API。
   - 策略实时运行状态查询。
3. **数据监控模块**：
   - 提供数据可视化所需的数据接口。
   - 支持 WebSocket 实现实时数据推送。
4. **任务管理模块**：
   - 定时任务调度。
   - 提供任务运行日志接口。
5. **告警模块**：
   - 触发规则报警，推送通知至网站与移动端。

---

### 2. **网站前端**
#### 技术选型
- 使用 React + TypeScript 实现组件化开发。
- 数据可视化选用 D3.js 或 ECharts。
- 配合 CSS 框架 Tailwind CSS 实现响应式布局。

#### 核心功能页面
1. **Dashboard（主控台页面）**：
   - 展示策略运行概况。
   - 主要数据在页面呈现：如收益率、回撤率等。
2. **策略管理**：
   - 策略的管理界面，包括查看任务进展、创建新任务。
3. **数据可视化页**：
   - 完成策略的运行后提供结果图表展示。
4. **告警中心**：
   - 收到来自后端的异常策略或任务提醒。
5. **用户设置**：
   - 用户个性化选项。
6. **登录/注册页面**：
   - 完成用户认证功能。

---

### 3. **移动端应用**
#### 技术选型
我们推荐两种技术选项：
1. **React Native** (推荐)：支持跨平台开发，并且开发生态与前端有一定重合，可减少学习成本。
2. **Flutter**：性能更优，适合对 UI 要求较高的场景。

#### 核心功能
1. **首页**：显示用户的待处理任务和策略运行概要。
2. **任务管理页面**：可启动、暂停或删除策略任务。
3. **策略监控**：显示策略详细的运行状态和结果。
4. **数据图表展示页面**：与前端网页功能相似。
5. **推送通知**：接收任务运行或异常告警。

---

### 4. **UI 设计**
#### 统一设计语言
采用 Material Design 或 Ant Design，整合：
- **颜色优化**：以蓝色为主色调构建设计风格。
- **图表样式一致性**：注重移动端和 PC 端体验统一。

#### 响应式设计
- 保证网页在各种尺寸屏幕上的正常展示。
- 对移动端实现更简洁的布局优化。

## 开发注意事项
1. **模块化开发**：
   - 保持前后端代码关注点分离，设计清晰的 API 调用接口。
2. **代码规范化**：
   - 后端遵循 PEP8，前端使用 ESLint/Prettier 保持代码风格一致。
3. **安全性检查**：
   - 后端增加输入验证、API 令牌。
   - 防范 XSS、CSRF 等前端常见威胁。
4. **持续集成/部署**：
   - 使用 GitHub Actions 实现 CI/CD。
   - 配置 Docker 容器化开发环境，保证部署一致性。

---

## 部署与运维
1. **容器化**：
   - 完成应用后，使用 Docker 容器化部署。
2. **云服务支持**：
   - 推荐使用 AWS、Azure 或 GCP 为后续提供横向扩展能力。
3. **监控与报警**：
   - 使用 Prometheus 或其他类似工具监控应用状态。
   - 关键异常时推送预警信息。
