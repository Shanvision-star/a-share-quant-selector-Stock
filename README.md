# a-share-quant-selector-Stock

> 🏆 **面向A股市场的专业级量化选股系统** — 多策略引擎 × 全市场扫描 × 专业Web工作台 × 钉钉自动推送

基于 **Python + akshare + FastAPI + Vue 3** 的A股量化选股系统，覆盖从数据获取、策略扫描、图形匹配到可视化分析的完整链路：

- **多策略量化引擎**：碗口反弹（BowlRebound）、B1完美图形匹配、B2三分类策略、阶段型B1前瞻扫描，多策略并行，按需组合。
- **全市场并发扫描**：自动按CPU核心数分配线程，单核设备自动降级，千只股票分钟级扫描完毕。
- **全自动风险过滤**：ST/*ST、退市、异常成交量、最大阴量出逃信号，多重过滤保障选股质量。
- **B1完美图形匹配**：基于12个历史成功案例，双线结构+量比+价格形态+KDJ四维相似度（DTW算法）精准排序。
- **B1阶段前瞻扫描**：基于掌阅科技模板抽象的6阶段模型，提前发现处于启动前窗口的潜在个股。
- **B2三分类策略**：横盘突破、灾后重建、平行重炮三类独立检测，便于按类型补案例与独立调参。
- **专业Web工作台**：Vue 3 + ECharts，支持K线交互分析、副图指标折叠/展开、公司信息面板、30日最大涨幅统计等。
- **钉钉自动推送**：选股结果 + K线图 + B1匹配排名自动推送，内置限流与指数退避重试。
- **通达信TXT导出**：一键生成可导入通达信的TXT文件，便于在软件中跟踪自选股。

## ✨ 核心功能

### 策略引擎
- 📈 **碗口反弹策略** — 基于「知行短期趋势线 > 知行多空线 + 放量阳线 + KDJ低位」三重共振的智能选股；结果自动标注回落碗中/靠近多空线/靠近短期趋势线三类
- 🎯 **B1完美图形匹配** — 基于12个历史成功案例，采用 **DTW曲线相似度算法**，从双线结构（30%）、量比特征（25%）、价格形态（25%）、KDJ状态（20%）四维度综合评分排序
- 🚀 **B1阶段前瞻扫描** — 以掌阅科技（603533）为模板，抽象出6阶段模型（低位蓄积→结构靠拢→缩量整理→局部放量→超卖确认→买点确认），扫描全市场处于 Stage 1-5 的潜在启动标的，**提前发现而非事后追涨**
- 🧱 **B2三分类策略** — 横盘突破、灾后重建、平行重炮三路独立实现，共用KDJ/趋势线/放量基础层，各类可独立调参和补案例，互不干扰

### 数据与扫描
- 🚫 **多重风险过滤** — ST/*ST、退市关键词、成交量异常、最大量为阴线（大资金出逃信号）四道过滤，开盘前自动剔除
- 🔄 **智能增量更新** — 15点前检查最近已收盘交易日，15点后检查当天数据；已是最新数据则直接跳过网络请求
- ⚙️ **自动并发/单核降级** — 默认按CPU核心数自动分配线程；检测到单核时自动切回单线程，避免线程切换拖慢执行；每只股票只读一次CSV，B1匹配与基础选股在同一工作线程内串行完成，消除重复IO

### Web工作台（Vue 3 + FastAPI）
- 🌐 **专业K线工作台** — Vue 3 + ECharts 渲染，主图含MA/BBI/短期趋势线/多空线，支持日线/周线切换，时间范围快速缩放（7日/30日/60日/1年/全部）
- 📐 **副图可拖拽与折叠** — 成交量、还原成交量、KDJ、MACD四个副图；分割线可拖拽调整高度；**默认折叠节省空间**，单击展开按钮或双击标题行一键展开放大
- 🔍 **内联分时K线弹窗** — 点击日K线任意一根K线，弹出可拖动的分时K线浮层（支持1分/15分切换），无需跳转页面
- 🏢 **公司信息面板** — 自动加载行业/地区/主营业务/概念板块/上市板等信息，辅助基本面判断
- 📊 **30日最大涨幅统计** — 在股票详情页直接展示近30个交易日的最大涨幅数据，快速评估历史弹性
- 📁 **通达信TXT一键导出** — 策略结果可直接生成通达信可导入的TXT格式，支持按策略和日期筛选下载

### 推送与通知
- 🔔 **钉钉自动推送** — 选股结果 + K线图 + B1匹配排名结果自动推送到钉钉群
- ⏱️ **智能限流保护** — 内置每分钟20条上限与2秒最小间隔，触发限速后自动指数退避重试（1s→4s→8s），避免触发钉钉错误码660026

```bash

git clone https://github.com/Shanvision-star/a-share-quant-selector-Stock
cd a-share-quant-selector-Stock

# 2. 创建并激活虚拟环境（推荐，避免依赖污染全局环境）
python3 -m venv .venv
source .venv/bin/activate          # Linux/macOS
.venv\Scripts\Activate.ps1        # Windows PowerShell

# 3. 安装依赖（Python后端）
pip install -r requirements.txt

# 4. 配置（可选）：填写钉钉 webhook 和 secret
# 编辑 config/config.yaml

# 5. 首次全量抓取历史行情数据（仅首次需要，已有数据可跳过）
python3 main.py init

# 6. 日常选股：自动更新数据 + 选股 + 钉钉推送
python3 main.py run

# 7. 带B1图形匹配的完整流程（推荐日常使用）
python3 main.py run --b1-match

# 8. 并发加速（按机器核心数自动扩展；单核自动降级保证稳定性）
python3 main.py run --b1-match --workers 10
```

## 📊 策略说明

### 碗口反弹策略 (BowlReboundStrategy)

#### 选股条件
1. **上升趋势** - 知行短期趋势线 > 知行多空线
2. **异动放量阳线** - 近期(M天内)存在放量阳线（成交量>=前日*N倍，且收盘价>开盘价）
3. **剔除最大阴量** - 回顾期内成交量最大的一天如果是阴线，直接剔除
4. **KDJ低位** - J值 <= 阈值，处于超卖区域

#### 分类标记（优先级从高到低）

| 分类 | 图标 | 条件 | 参数 |
|------|------|------|------|
| **回落碗中** | 🥣 | 价格位于知行短期趋势线和知行多空线之间 | - |
| **靠近多空线** | 📊 | 价格距离知行多空线 ±duokong_pct% | `duokong_pct`: 默认3% |
| **靠近短期趋势线** | 📈 | 价格距离知行短期趋势线 ±short_pct% | `short_pct`: 默认2% |

#### 案例12（掌阅科技 603533）—— 详细特征解析

下面把 README 中使用的 11 项特征（双线结构 / 量比 / 价格形态 / KDJ 等）逐项对应回溯到掌阅科技的案例上，并给出阶段化的观察要点，便于把该案例作为 B1 阶段模板使用：

- **整体阶段（Stage 1-6）时间线**：
  - Stage1（低位蓄积）: 成交量逐步萎缩，价格在多空线下方或靠近多空线波动；
  - Stage2（结构靠拢）: 短期趋势线与多空线逐步靠拢，双线斜率趋于平缓；
  - Stage3（缩量整理）: 价格进入平台或小幅震荡，量比明显低于历史均值（缩量）；
  - Stage4（局部放量）: 出现放量阳线，成交量 >= 前日 N 倍（N 可由策略参数 `N` 控制），且收盘价站上短期趋势线或多空线上方；
  - Stage5（确认超卖与回撤）: J 值处于低位或发生金叉，最大量不是阴量（通过最大阴量过滤保留强势筹码流入信号）；
  - Stage6（买点确认）: 突破或确认信号后在 2026-02-09 被系统判定为买点。

- **双线结构（对应 README 中的双线维度）**：
  - 案例显示 `知行短期趋势线` 在突破前与 `知行多空线` 呈现“靠拢然后发散”的路径，斜率在突破前出现抬升，符合“双线结构”权重较高的匹配特征。

- **量比特征（成交量）**：
  - 在 Stage3 期间明显缩量；随后 Stage4 出现放量阳线，且放量幅度满足 `成交量>=前日*N倍` 的放量阈值（策略默认 `N` 在 `config/strategy_params.yaml` 可调），且最大量日为阳线而非阴线。

- **价格形态（形态相似度）**：
  - 价格在回撤整理阶段形成“碗口/平台”轮廓，归一化后的价格曲线与案例库中掌阅科技模板在 DTW 或曲线相似度上得分较高（属于 B1 高相似度样式）。

- **KDJ / J 值状态**：
  - 整体进入选股区间时 J 值处于低位或呈下探后回升（或金叉），满足系统对“超卖后回升”的偏好（`J_VAL` 参数驱动）。

- **最大阴量过滤**：
  - 回顾期内最大成交量对应的交易日为阳线（非阴量），因此未被最大阴量过滤机制剔除，这对案例被保留起了关键作用。

- **回撤与短期趋势线关系**：
  - 在确认阶段，价格不会大幅跌破短期趋势线（回撤在允许的 `trend-drop-pct` 范围内），且在突破或确认日回补或站上该短期线。

- **分类标签（系统输出）**：
  - 案例在被选出时被标注为“阶段型 B1 setup 窗口”，同时根据价格与两条线的位置可归类为“回落碗中”或“靠近短期趋势线”。

- **应用建议（参数与检验）**：
  - 使用该案例作为模板时建议：`lookback_days` 取 20~30 日以覆盖完整整理段；`min_similarity` 取 60~75 以平衡命中数与质量；`N`（放量倍数）保守取 2~3，根据市场活跃度放宽或收紧。

- **示例关键事件点（便于回测/复盘定位）**：
  - 2026-02-06：系统将该标的识别为进入 B1 setup 窗口的起点；
  - 2026-02-09：系统判定买点（符合放量/收盘站稳条件并满足 J 值与双线结构要求）。

以上逐项把 README 中使用的“11 个特征”与 `掌阅科技（603533）` 案例具体事件关联起来，便于把该案例作为 B1 阶段模板进行批量扫描时的参数校准与回溯验证。

### B2 三分类策略

当前 `strategy/b2_strategy.py` 已不再只按单一路径做 B2 检测，而是拆成 3 个可独立扩展的分类入口：

1. **横盘突破型**：重点识别“平台整理 -> B1 低位确认 -> 放量突破平台高点”的标准结构，代表案例是星环科技。
2. **灾后重建型**：重点识别“前期存在明显破坏段 -> 放量反转长阳 -> 修复平台 -> 再突破”的修复结构。
3. **平行重炮型**：重点识别“多根相近价位大阳反复攻击 -> 中间缩量回踩 -> B1 过渡 -> 再放量启动”的进攻结构。

这三类共用 KDJ、知行短期趋势线、知行多空线、放量突破和站稳多空线等基础条件，但在“前置背景”和“整理区识别”上已经分开实现，后续补案例时可以继续按分类扩展，而不是把所有逻辑堆进一个函数里。
### B1阶段策略（前瞻扫描）

B1阶段策略用于“启动前预警”，不是只做事后复盘。

- 固定模板（603533）用于案例验证：确认6阶段完整结构。
- 动态扫描用于全市场选股：重点筛出阶段1-5成立、阶段6待确认的标的。
- 时间口径全部按交易日计数（不是自然日）。

📖 **[查看阶段策略详解 →](B1_STAGE_STRATEGY.md)**

#### 匹配维度（三维相似度）

| 维度 | 权重 | 说明 |
|------|------|------|
| **双线结构** | 30% | 知行短期趋势线与多空线的相对位置、斜率、发散程度 |
| **量比特征** | 25% | 成交量变化模式（缩量后放量、持续放量等） |
| **价格形态** | 25% | 归一化价格曲线相似度（DTW算法） |
| **KDJ状态** | 20% | J值位置、金叉状态、趋势方向 |

#### 输出结果

匹配结果按相似度从高到低排序，钉钉通知包含：
- 每只股票的相似度百分比
- 匹配的历史案例名称和日期
- 分项得分（双线/量比/形态/KDJ）
- 策略分类、价格、J值等信息

#### 风险过滤说明

**自动剔除的股票类型：**

| 过滤条件 | 说明 |
|----------|------|
| **ST/*ST股票** | 名称以 ST 或 *ST 开头的股票 |
| **退市风险股** | 名称包含"退"、"退市"、"已退"等关键词 |
| **数据异常股** | 成交量为0、收盘价异常、J值均值超过80的股票 |
| **最大量是阴量** | 回顾期内成交量最大的一天是阴线（新增强力过滤）|

> **关于"最大量是阴量"过滤**：
> - 如果在 M 天回顾期内，成交量最大的一天是阴线（收盘价 < 开盘价），则该股票会被直接剔除
> - 这通常意味着大资金在出逃，属于弱势信号，剔除后可提高选股质量

#### 技术指标定义

**知行短期趋势线**
```
EMA(EMA(CLOSE, 10), 10)
```
对收盘价连续做两次10日指数移动平均

**知行多空线**
```
(MA(CLOSE, 14) + MA(CLOSE, 28) + MA(CLOSE, 57) + MA(CLOSE, 114)) / 4
```
四条均线平均值

#### B1 运行提速说明

`python3 main.py run --b1-match` 现在做了两层提速优化：

1. **基础选股改为单股一次读盘**：同一只股票只读取一次 CSV，在同一个工作线程里连续跑 BowlReboundStrategy 和 B1CaseStrategy，避免以前按策略重复读盘、重复算指标。
2. **并发模式自动判定**：默认根据 CPU 核心数自动分配线程；如果机器只有单核，程序会自动回退到单线程，避免线程切换成本反而拖慢运行。

如果你在低配机器上仍然觉得慢，可以优先调这几个参数：

1. `--max-stocks 500`：先缩小扫描范围验证逻辑。
2. `--workers 4`：在 4 核以上机器上显式限制线程，避免开太多线程导致磁盘竞争。
3. `config/strategy_params.yaml` 里的 `B1PatternMatch.max_candidates`：降低 B1 二次匹配的候选上限。

### 三、关键参数注释（与代码参数一致）

| 参数 | 作用 | 默认值 |
|------|------|--------|
| `--max-stocks` | 限制处理股票数量（用于快速测试） | `None`（不限制，全市场） |
| `--workers` | 并发线程数（选股/匹配加速） | `None`（自动按系统策略分配） |
| `--category` | 分类过滤（`all/bowl_center/near_duokong/near_short_trend`） | `all` |
| `--b1-match` | 启用 B1 完美图形匹配 | `False`（不传则关闭） |
| `--lookback-days` | B1 匹配回看交易日天数 | `None`（读取配置默认 `25`） |
| `--min-similarity` | B1 最低相似度阈值（百分比，0-100） | `None`（读取配置默认 `60`） |
| `--backtest-days` | 回溯天数（连续 K 条件窗口） | `3` |
| `--k-threshold` | 回溯 K 值阈值 | `20.0` |
| `--trend-drop-pct` | 短期趋势线最大容忍回落百分比（如 5=最多回落5%） | `5.0` |
| `--host` | Web 服务监听地址 | `0.0.0.0` |
| `--port` | Web 服务端口 | `5000` |
| `--config` | 指定配置文件路径 | `config/config.yaml` |

### 四、参数含义示例（天数 / 百分比）

- `lookback_days=25`：向前取最近 25 个交易日做 B1 形态匹配，不含周末和休市日。
- `min_similarity=60`：最低相似度 60%，即综合得分低于 60% 的股票会被过滤。
- `min_similarity=70`：更严格阈值，命中数量通常下降，但形态一致性更高。
- `trend-drop-pct=5`：回溯筛选时，股价相对短期趋势线最多允许下探 5%。
- `duokong_pct=3`：分类“靠近多空线”时，要求股价与多空线偏离在 ±3% 内。
- `short_pct=2`：分类“靠近短期趋势线”时，要求股价与短期趋势线偏离在 ±2% 内。

### 五、配置文件参数位置

编辑 config/strategy_params.yaml 中 B1PatternMatch 可调整：
- min_similarity: 默认最小相似度阈值
- lookback_days: 默认回看天数
- weights: 四维相似度权重
- tolerances: 各项特征容差

### 智能更新逻辑

`python3 main.py run` 会自动判断是否需要更新数据：

1. **15点前** - 检查本地是否已同步到最近已收盘交易日
2. **15点后** - 检查每只股票是否有当天数据
3. **如果本地已有最新交易日数据** - 跳过网络更新，直接使用
4. **否则** - 执行增量更新

## ⏱️ 钉钉限流保护

系统内置智能限流器，防止触发钉钉 API 频率限制（错误码 660026）：

| 限制项 | 默认值 | 说明 |
|--------|--------|------|
| 每分钟最大消息数 | 20 条 | 达到限制后自动等待 |
| 最小发送间隔 | 2 秒 | 每条消息间隔至少 2 秒 |
| 重试次数 | 3 次 | 遇到限速错误时指数退避重试 |

**退避策略**：
- 第 1 次重试：等待 1 秒
- 第 2 次重试：等待 4 秒  
- 第 3 次重试：等待 8 秒

## 🏅 项目亮点总结

| 亮点 | 说明 |
|------|------|
| **多策略并行，按需组合** | BowlRebound / B1匹配 / B2三分类 / 阶段前瞻四套策略独立运行，`--b1-match` 等参数自由组合 |
| **DTW图形相似度** | 采用动态时间规整（DTW）算法计算归一化价格曲线相似度，比简单欧氏距离更能识别时间轴上的形态变形 |
| **B1阶段前瞻而非追涨** | 6阶段模型在买点确认前1-3个交易日给出预警，让系统在启动前布局而非启动后跟进 |
| **SSE实时进度流** | 数据更新和策略重建通过 Server-Sent Events 将进度、阶段、当前股票代码、命中信号实时推送到前端，无需轮询 |
| **副图默认折叠** | K线工作台打开时副图默认折叠，主图占80%高度，减少干扰；需要分析时双击标题一键展开 |
| **内联分时弹窗** | 单击日K线任意一根蜡烛图即可在主图上弹出可拖动的分时K线浮层，避免页面跳转打断分析思路 |
| **公司信息懒加载** | `/stock/info` 接口按需拉取，前端加缓存，不影响K线主图首屏加载速度 |
| **通达信TXT无缝导出** | 选好的股票一键生成通达信TXT，导入后直接在软件中设置提醒或查看K线，连接量化系统与桌面软件 |
| **一键启动脚本** | `start_dev.bat` 自动打开后端+前端两个终端窗口，开发环境0配置启动 |
| **全平台兼容** | Linux/macOS/Windows 均有完整操作步骤，虚拟环境隔离，依赖一行安装 |
