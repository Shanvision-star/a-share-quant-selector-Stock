# B1 阶段型策略说明

> 本文件用于描述「掌阅科技」类型的 B1 完美图形识别逻辑，方便后续回溯优化时对照代码参数。  
> 对应代码：`strategy/b1_case_analyzer.py` · `strategy/pattern_config.py` → `B1_STAGE_CASES`

---

## 整体节奏（以掌阅科技为参考）

```
2025-10-17  支撑低点（主力防守底线）
    ↓
2025-11-17  ① 低位KDJ触发（J ≤ 13，主力建仓）
    ↓
2025-11-18~20  ② 护盘确认（3天不破11/17开盘价）
    ↓
2025-11-21  ③ 大阳突破 + 连续3天阳线（正式启动）
    ↓
2025-11-26~12-17  ④ 洗盘16个交易日（守住10/17低点不跌破）
    ↓
2026-02-04~06  ⑤ 回踩知行多空线（setup窗口，J ≤ 20，偏离 ≤ ±3%）
    ↓
2026-02-09  ⑥ 大阳买点确认（涨幅 ≥ 9.5%，突破近5日高点）
```

## 六个阶段详解

### 阶段一 · 低位 KDJ 触发（anchor_kdj）

| 参数                        | 值         |
| --------------------------- | ---------- |
| 触发日期 `anchor_date`      | 2025-11-17 |
| 判断字段 `anchor_kdj_field` | J          |
| 门槛 `anchor_kdj_max`       | ≤ 13       |
| 掌阅科技实际值              | J = 1.73   |

**含义**：KDJ 的 J 值下行到极度超卖区间（≤ 13），是主力完成压价建仓的信号。这是整段结构的起点，不在此日触发后续阶段均不成立。

---

### 阶段二 · 护盘确认（guard_days）

| 参数                           | 值                  |
| ------------------------------ | ------------------- |
| 观察天数 `no_break_days`       | 3 个交易日          |
| 参考价 `guard_reference_field` | 触发日开盘价 (open) |
| 比较字段 `guard_compare_field` | 每日最低价 (low)    |

**含义**：触发日后连续 3 个交易日，每天的最低价都不能跌破触发日的开盘价。  
3 天内没有跌破 → 主力在护盘，筹码锁定，散户割肉意愿弱。  
任意一天跌破 → 结构不成立，整段过滤掉。

---

### 阶段三 · 大阳突破 + 连涨（breakout_streak）

| 参数                               | 值                     |
| ---------------------------------- | ---------------------- |
| 突破日期 `breakout_date`           | 2025-11-21             |
| 单日最低涨幅 `breakout_min_pct`    | ≥ 4.0%                 |
| 连续阳线天数 `breakout_min_streak` | ≥ 3 天                 |
| 掌阅科技实际表现                   | 11/21大阳，随后3天连涨 |

**含义**：主力完成建仓后拉升去消耗上方浮筹。大阳线当天涨幅不低于 4%，且连续保持阳线至少 3 天。连涨动能越强，说明主力拉升意志越坚决。

---

### 阶段四 · 洗盘守住支撑低点（washout_support）

| 参数                              | 值                           |
| --------------------------------- | ---------------------------- |
| 支撑低点日期 `reference_low_date` | 2025-10-17                   |
| 支撑低点价格（比较基准）          | 2025-10-17 的最低价 18.14 元 |
| 洗盘结束日期 `washout_end_date`   | 2025-12-17                   |
| 比较字段 `washout_compare_field`  | 收盘价 (close)               |
| 保护逻辑                          | 洗盘段所有收盘价 ≥ 18.14 元  |
| 掌阅科技洗盘天数                  | 16 个交易日                  |
| 掌阅科技洗盘最低收盘              | 18.66 元                     |
| 掌阅科技洗盘最低价                | 18.51 元                     |

**含义**：这是整段结构中「主力控盘」验证最核心的一条。  
主力绝不允许洗盘期间的收盘价跌回上一个低点（18.14 元），说明：

- 主力手中筹码充足，能在关键支撑位护盘；
- 散户割肉的低点被主力接走，不给二次低位买入机会；
- 股价守住支撑低点 → 结构继续有效；
- 任意一天收盘跌破支撑低点 → 结构作废。

---

### 阶段五 · 回踩知行多空线 Setup 窗口（revisit_signal）

| 参数                              | 值                                |
| --------------------------------- | --------------------------------- |
| 回踩结束日期 `revisit_date`       | 2026-02-06                        |
| 观察窗口天数 `setup_window_days`  | 3 个交易日（至少 2 天满足即通过） |
| 多空线偏离阈值 `revisit_band_pct` | ±3.0%                             |
| KDJ 判断字段 `revisit_kdj_field`  | J                                 |
| KDJ 门槛 `revisit_kdj_max`        | ≤ 20.0                            |
| 掌阅科技 2026-02-06 偏离          | +0.66%                            |
| 掌阅科技 2026-02-06 J 值          | 14.59                             |

**含义**：经过约 33 个交易日的震荡整理后，股价再次回落到知行多空线附近（±3% 以内），J 值同步回落到低位（≤ 20）。  

- 连续 3 个交易日都满足：偏离 ≤ ±3% 且 J ≤ 20 → setup 窗口确认；
- 这个状态是「观察预警区间」，不是买点；
- 说明主力完成了二次换手布局，准备再次启动。

**知行多空线定义**：对应代码中 `bull_bear_line`，使用 M3=57、M4=114 双均线的中轴位置。

### 阶段六 · 大阳买点确认（buy_signal）

| 参数                                      | 值                                 |
| ----------------------------------------- | ---------------------------------- |
| 买点日期 `buy_date`                       | 2026-02-09                         |
| 涨幅基准 `buy_compare_field`              | 前一日收盘价 (close)               |
| 最低涨幅门槛 `buy_trigger_min_pct`        | ≥ 4.0%                             |
| 突破回看天数 `buy_breakout_lookback_days` | 近 5 个交易日最高价                |
| 掌阅科技 2026-02-09 涨幅                  | 约 10.02%（23.35 → 25.69 元）      |
| 一字涨停判断                              | 开=收=高=低 则标记 limit_like=True |

**含义**：setup 窗口结束后出现的第一根大阳线，且：

1. 相对前一日收盘涨幅 ≥ 9.5%；
2. 当天收盘价突破近 5 个交易日的最高价；
3. 买点日期必须晚于 setup 窗口结束日。

三个条件同时满足 → 阶段型 B1 完整成立 → 结构 passed=True。

## 推荐回溯天数

| 口径           | 计算方式                      | 掌阅科技实际值 |
| -------------- | ----------------------------- | -------------- |
| anchor → buy   | 从 2025-11-17 算到 2026-02-09 | 59 个交易日    |
| support → buy  | 从 2025-10-17 算到 2026-02-09 | 80 个交易日    |
| anchor → setup | 从 2025-11-17 算到 2026-02-06 | 57 个交易日    |

> 默认使用 `lookback_days = 80`（`support → buy` 口径），确保支撑低点一并纳入分析窗口。

## 时间口径说明（交易日）

- 本策略中的“天数”默认均为交易日，不是自然日。
- 固定日期（例如 `2025-11-17`、`2026-02-06`、`2026-02-09`）均指对应交易日的 K 线。
- 连续天数判断（如“连续 3 天不破”“连涨 3 天”）均按连续交易日计算。
- 若个股存在停牌/缺失数据，程序按数据中可用的连续 K 线记录计算。

## 适用范围与限制

- 阶段型 B1 分析器（`B1CaseAnalyzer`）是**独立模块**，以增量方式补充到现有 B1 体系中，不改动原有碗口反弹策略和普通 B1 相似度匹配逻辑。
- 阶段型 B1 有两个用途：
  1. **案例模板验证**：用掌阅科技这条已知成功走势做固定日期复盘，确认“完整阶段型 B1”长什么样。
  2. **前瞻选股预警**：把掌阅科技案例抽象成动态规则，在全市场扫描“还没正式启动、但已经进入 setup 区”的候选股。
- 掌阅科技模板只对股票代码 `603533` 执行固定日期阶段匹配，全市场扫描时非目标股票直接跳过，不会按掌阅科技的历史日期去硬匹配其他股票。
- 这并不代表阶段型 B1 不能选股。不能直接全市场复用的是**掌阅科技固定日期模板**；真正用于选股的是从该案例抽象出来的**动态阶段规则**。
- 动态前瞻选股的目标不是去找“已经走出 2026-02-09 大阳”的股票，而是去找“阶段一到阶段五已经基本成立、阶段六大阳确认还没出现”的预备标的。
- 换句话说：
  - **模板**负责回答“掌阅科技为什么是标准阶段型 B1”。
  - **规则**负责回答“今天市场里谁正在长成类似掌阅科技的结构”。
- 洗盘守住支撑低点是**一票否决**条件：只要洗盘期间任意一天收盘价跌破参考低点，整个结构判定为 `passed=False`。
- 本策略描述的是**历史案例复盘口径**，实盘应用时需结合当前市场量价自行判断是否具备相同的主力控盘特征。

## 实盘选股口径

### 1. 预警信号是什么

- 实盘扫描时，阶段型 B1 不是等到阶段六大阳出现后才提示，而是优先寻找**阶段一到阶段五已经基本成立**的股票。
- 更具体地说，程序会重点找这类标的：
  - 前面已经出现过一次低位 KDJ 触发；
  - 后面走出过护盘和突破连涨；
  - 洗盘阶段没有明显破坏支撑；
  - 当前股价正在知行多空线附近整理，且 KDJ 再次回到低位。
- 这类股票属于“**预备标的**”，含义是结构已经接近完成，但还没有正式启动。

### 2. 什么情况下可以重点观察

- 当股票进入 setup 窗口时，可以进入重点观察名单。
- 当前项目里的 setup 口径是：
  - 股价距离知行多空线在 ±3% 以内；
  - J 值不高于 20；
  - 在最近 2 个交易日里，至少有 1 天满足上述条件。
- 这一步的本质不是确认买点，而是确认“主力二次洗盘后，股价重新回到低风险区附近”。

### 3. 什么情况下才算确认买点

- 真正的确认买点，仍然是 setup 之后出现的第一根强启动大阳线。
- 当前文档里的确认口径是：
  - 相对前一日收盘涨幅不低于 9.5%；
  - 收盘价突破近 5 个交易日高点；
  - 出现时间晚于 setup 窗口结束日。
- 这一步更适合做“确认加仓”或“强势跟随”，不适合作为唯一的提前埋伏条件。

### 4. 为什么不能只等阶段六

- 如果等到阶段六大阳才认定为 B1，通常价格已经明显抬升，容易错过低风险买点。
- 所以阶段六更适合做**案例验证**和**信号确认**，而不是唯一的选股入口。
- 对实盘更有价值的是阶段一到阶段五构成的“预警结构”，因为这时股票还处在启动前或刚准备启动的位置。

### 5. 更合理的实盘使用方式

- 第一层：用动态阶段规则筛出“阶段一到阶段五基本成立”的预警股。
- 第二层：结合量能、盘口、所属板块强弱和当天分时表现，判断是否值得提前埋伏。
- 第三层：如果随后出现阶段六大阳，则视为强确认信号；如果始终不启动，则继续跟踪或放弃。

### 6. 什么情况下要直接放弃

- 洗盘阶段任意一天收盘跌破参考支撑低点，直接放弃。
- setup 阶段虽然靠近多空线，但 KDJ 无法重新回到低位，说明整理质量不够，放弃。
- 股价跌破关键支撑 5% 以上时，要高度怀疑主力控盘失败，而不是把它继续当作标准 B1。

### 7. 对掌阅科技案例的落地理解

- 2026-02-06 更适合作为“进入重点观察区”的日期，因为它对应的是回踩多空线后的低风险 setup 状态。
- 2026-02-09 更适合作为“正式确认买点”的日期，因为这一天已经是强启动大阳。
- 所以掌阅科技案例告诉我们的不是“必须等涨停才算 B1”，而是：
  - **2 月 6 日附近是可埋伏的观察点**；
  - **2 月 9 日是结构被市场确认的启动点**。

## 扫描输出与结果落地（已实现）

### 1. 扫描进度显示

- `run_b1_scan.py` 已支持实时进度输出。
- 进度项包括：已处理/总数、百分比、速度（只/秒）、ETA、命中数、异常数。
- 支持心跳输出与强制刷新（`flush=True`），避免终端长时间无反馈。

### 2. 命中结果展示

- 命中列表已显示为“股票代码 + 中文名称”，不再只显示纯数字代码。
- 同步展示关键字段：anchor、setup、curJ、多空线偏离、支撑价。

### 3. TXT 导出规则（通达信观察）

- 阶段型 B1 命中结果导出目录：`data/txt/B1-zykj-match/`
- 文件名包含策略名称和选股日期时间，便于回溯管理。
- 导出文件用于通达信导入观察，同时保留名称信息便于人工核对。

### 4. 钉钉推送行为

- 触发条件：有命中结果 + webhook 配置有效。
- 无命中时默认不推送，避免空结果刷屏。
- 发送流程已增强日志：会明确打印成功/失败，便于定位“未触发”与“发送失败”的区别。

### 5. 钉钉未推送排查清单

1. webhook 未配置或配置为空。
2. 本次扫描命中数为 0（策略按设计不推空消息）。
3. 网络或权限问题导致 webhook 请求失败。
4. 平台限流导致请求被拒绝。
