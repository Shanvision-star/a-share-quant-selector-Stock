# B2 完美图形匹配详解

本文档详细解释 B2 完美图形匹配的相似度计算逻辑，参考并扩展自 B1_PATTERN_MATCH.md。

---

## 概述

B2 完美图形匹配是一种**两阶段选股框架**：

1. **规则扫描**（B2PatternLibrary）：6步严格条件过滤全市场，找出符合 B2 突破形态的股票
2. **图形匹配**（B2PatternMatchLibrary）：对规则命中的候选股，计算其与历史成功案例的多维相似度，并按相似度排序

**核心目标**：在已满足 B2 规则的候选池中，优先推荐形态结构与历史成功案例最相似的股票，提高胜率。

---

## 与 B1 图形匹配的关键区别

| 对比维度 | B1 图形匹配 | B2 图形匹配 |
|---------|------------|------------|
| **候选来源** | BowlReboundStrategy 选出的股票 | B2 规则扫描（6步逻辑）命中的股票 |
| **特征窗口** | B1 突破日前 25 天 | B2 突破日前 40 天（覆盖整理期+B1期） |
| **量能权重** | 25% | **35%**（缩量整理是 B2 核心特征） |
| **价格权重** | 25% | **15%**（B2 价格形态多样，降低依赖） |
| **案例类型** | 单一类型（低位碗型整理） | **三分类**（横盘突破/灾后重建/平行重炮） |
| **默认阈值** | 60% | 55%（B2 形态多样，阈值略低） |

## B2 三分类形态说明

### 横盘突破型（sideways_breakout）

**形态特征**：
- 大阳线攻击后，在相对稳定价格区间内横盘整理
- 整理期成交量持续萎缩（主力锁仓）
- B1 日 J 值创低位，价格贴近短期趋势线（踩线不破）
- B2 突破日：放量大阳突破整理区间最高价（"盖板价"）

**代表案例**：星环科技 688663 （2025-12-15 突破）

---

### 灾后重建型（post_crash_rebuild）

**形态特征**：
- 前期有明显大幅下跌段（回撤 ≥ 18%），称为"破坏段"
- 破坏段结束后出现放量反转长阳（涨幅 ≥ 6%），开始"灾后修复"
- 在修复平台上出现 B1 信号（J 值低位 + 趋势向上）
- B2 突破日：放量突破修复平台高点，"重建完成"

**代表案例**：晶科科技 601778 （2025-08-07 突破）

---

### 平行重炮型（parallel_artillery）

**形态特征**：
- 多根（2-4 根）收盘价落在相近价格带（带宽 ≤ 2%）的大阳线
- 多根大阳线之间穿插缩量小阴/小阳（"红肥绿瘦"）
- 最后一根平行大阳到 B1 信号间隔 ≤ 15 个交易日
- B2 突破日：再次放量突破，完成第三波攻击

**代表案例**：四会富仕 300852、百普赛斯 301080

---

## 匹配维度与权重

| 维度 | 权重 | 说明 |
|------|------|------|
| **量能特征** | **35%** | 成交量变化模式（缩量整理+放量突破），B2 最核心特征 |
| **双线结构** | 25% | 知行短期趋势线与多空线的相对位置、斜率 |
| **KDJ 状态** | 25% | J 值区域（必须经历低 J 值），反映调整充分程度 |
| **价格形态** | 15% | 归一化价格曲线形状（DTW 算法），B2 形态多样化权重较低 |

---

## 特征窗口提取逻辑

**B2 特征窗口 = B2 突破日前 40 个交易日（不含突破当天）**

```
B2突破日（不含）
   ↑
   40个交易日  <——  特征提取窗口
   ↓
   │  整理平台期（波动率低，缩量整理）
   │  B1 信号期（J 值触底，量能最小）
   │  大阳线期（放量攻击，形态蓄力）
   │
   最早一天（t-40）
```

**为什么排除突破当天？**

B2 突破当天是放量大阳线，会让以下特征严重失真：
- 量能比：单日放量会导致均量比虚高
- 价格偏离：收盘超越趋势线，破坏"贴线"特征
- KDJ：突破当天 J 值上升，不反映"超卖启动"状态

---

## 相似度计算详解

### 1. 量能特征相似度（35%）

**核心思想**：B2 的"缩量整理 + 放量突破"是最可靠的量价结构，权重最高。

| 特征 | 计算方法 | 含义 |
|------|----------|------|
| `avg_volume_ratio` | 近10日均量 / 前10日均量 | 近期量能变化趋势 |
| `shrink_then_expand` | 前半段缩量且后半段放量 | 缩量后放量模式 |
| `volume_trend` | 线性回归分类 | 量能整体趋势 |
| `max_volume_ratio` | 单日最大量比 | 最大放量倍数 |

```python
# 均量比相似（允许 ±1.5 倍差异）
ratio_diff = abs(cand_ratio - case_ratio)
sim = max(0, 1 - ratio_diff / 1.5)

# 缩量后放量模式一致性
if 同为缩量后放量:
    sim = 1.0
else:
    sim = 0.5
```

---

### 2. 双线结构相似度（25%）

与 B1 相同，比较知行趋势线的相对位置（百分比偏离，消除价格绝对值影响）。

| 特征 | 含义 |
|------|------|
| `short_vs_bullbear` | 短期趋势线/多空线 |
| `short_slope` | 短期趋势线斜率方向 |
| `is_in_bowl` | 价格是否在"碗中"（短趋势 > 价格 > 多空线） |
| `price_bias_pct` | 价格相对双线均值的百分比偏离 |

---

### 3. KDJ 状态相似度（25%）

**B2 特有关注点**：B2 突破前必须经历 J 值低位（< 20），否则可能是高位追涨信号。

| 特征 | 含义 |
|------|------|
| `j_value` | J 值绝对数值 |
| `j_position` | 低位（≤20） / 中位 / 高位（≥80） |
| `k_cross_d` | 是否发生 KDJ 金叉 |
| `j_rebound` | J 值近 3 日回升趋势 |

```python
# J 值位置一致性（最重要）
if 同为低位:
    sim = 1.0
elif 低位 vs 中位:
    sim = 0.7
else:
    sim = 0.4
```

---

### 4. 价格形态相似度（15%）

使用 DTW（动态时间规整）算法比较归一化价格曲线形状，与 B1 逻辑相同，但权重降至 15%。

```python
# 归一化（0-1范围）
curve = (closes - min_price) / (max_price - min_price)

# DTW 相似度
distance, _ = fastdtw(cand_curve, case_curve, dist=euclidean)
sim = max(0, 1 - distance / max(len(cand), len(case)))
```

---

## 阈值说明

| 相似度 | 含义 | 建议操作 |
|--------|------|--------|
| ≥ 80% | 形态高度相似 | 重点关注，优先研究 |
| 65-79% | 较为相似 | 值得关注，辅助决策 |
| 55-64% | 一般相似 | 参考价值，结合其他指标 |
| < 55% | 形态差异大 | 不显示（默认过滤） |

## 使用方法

### 命令行

```bash
# B2 完美图形匹配（规则扫描 + 相似度打分，默认阈值 55%）
python main.py run --b2-pattern-match

# 调整相似度阈值（严格模式）
python main.py run --b2-pattern-match --min-similarity 65

# 限制股票数量（调试用）
python main.py run --b2-pattern-match --max-stocks 300

# 仅规则扫描（不做相似度匹配）
python main.py run --b2-match
```

## 输出内容

### 1. 通达信导入 TXT

位置：`data/txt/B2-match/tdx_B2_matched_YYYYMMDD_HHMM.txt`

格式：
```
SH688663
SZ300852
SZ301080
...
```

### 2. 详细注释 TXT（含相似度）

位置：`data/txt/B2-match/B2图形匹配结果_YYYYMMDD_HHMM.txt`

包含内容：相似度总分、四维分项、B1/B2 日期、整理区间、止损价、匹配案例

### 3. 钉钉通知

**汇总消息**：命中总数 + 三分类统计 + 相似度排行榜

**个股详情**（逐只发送）：
- 相似度总分 + 四维分项
- 形态类型 + 匹配历史案例
- B1/B2 日期、涨幅、量比
- 整理区间高点
- **止损价**（B2 突破日最低价）

**K 线图**：标注 B1 和 B2 日期，展示最近 60 个交易日走势

## 注意事项

1. **首次运行**：系统自动从本地 CSV 数据构建案例特征库，并缓存到 `data/b2_pattern_match_cache.json`
2. **案例数据**：需要本地有以下股票的 CSV 数据：
   - 688663（星环科技）
   - 601778（晶科科技）
   - 300852（四会富仕）
   - 301080（百普赛斯）
3. **缺失案例**：若某个案例的 CSV 不存在，系统会跳过该案例（不会中断）
4. **缓存失效**：修改 `B2_PATTERN_CASES` 配置后，系统会自动重建缓存
5. **相似度 ≠ 必然上涨**：历史数据不代表未来，请结合基本面和市场环境综合判断

## 扩展案例

可在 `strategy/pattern_config.py` 的 `B2_PATTERN_CASES` 中追加案例：

```python
B2_PATTERN_CASES.append({
    "id": "b2p_case_005",
    "name": "股票名称",
    "code": "XXXXXX",
    "b2_date": "YYYY-MM-DD",    # B2 突破日
    "lookback_days": 40,
    "pattern_type": "sideways_breakout",  # 或 post_crash_rebuild / parallel_artillery
    "tags": ["板块", "特征"],
    "description": "案例简要说明",
})
```

追加后，清除 `data/b2_pattern_match_cache.json`（或修改会自动触发重建），下次运行时自动加载新案例。
