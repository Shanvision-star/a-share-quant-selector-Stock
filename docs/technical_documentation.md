# 技术说明文档

## 项目概述
A-Share Quant Selector 是一个量化选股系统，旨在通过多种策略分析股票数据，生成选股结果，并通过 Web 界面展示。

### 当前功能模块
1. **策略分析接口**：
   - 使用 FastAPI 实现的后端接口。
   - 支持多种策略（如 B1/B2 策略）。
   - 提供离线缓存机制以优化性能。

2. **前端界面**：
   - 使用 Vue 3 和 Vite 构建。
   - 集成 ECharts 实现 K 线图表。
   - 提供股票详情页和选股结果页。

3. **数据更新**：
   - 支持通过脚本更新股票数据。
   - 使用 JSON 文件存储缓存数据。

4. **通知功能**：
   - 集成 DingTalk 推送通知。

## 核心文件说明

### 后端
1. **`strategy_service.py`**
   - 实现策略分析逻辑。
   - 提供 RESTful API 接口。
   - 核心函数：`_scan_strategy`、`run_strategy`、`build_strategy_result_snapshot`。

2. **`generate_web_strategy_results.py`**
   - 用于生成离线策略缓存。
   - 当前状态：脚本已执行，但需验证输出文件。

3. **`web_server.py`**
   - FastAPI 主应用入口。

### 前端
1. **`HomeView.vue`**
   - 展示选股结果。
   - 支持过滤和分页。

2. **`StockDetail.vue`**
   - 展示单只股票的详细信息。
   - 集成 K 线图表。

## 2026-04 架构更新（参数保存 / 前端加载 / 列表编排）

### 1) 参数配置接口与运行时一致性
- `GET /api/config` 返回 `{ revision, updated_at, configs }`。
- `POST /api/config` 采用乐观锁（`expected_revision`），冲突返回 `409`。
- 保存成功后会刷新策略运行时参数并清理相关缓存，避免“保存后仍读取旧参数”。

### 2) 前端状态与请求控制
- 设置页改为“草稿 + 手动保存”模型：未保存离开会提示确认。
- 首页/策略结果页引入共享查询状态与请求管理（去重、取消、latest-request-wins）。
- K 线请求支持 `AbortSignal`，切换目标时中止过期请求，减少图表无效计算。

### 3) 股票列表后端编排下沉
- 列表过滤、排序、分页、快照分支逻辑从 Router 下沉到 `stock_list_service.py`。
- 路由层通过 threadpool 调用同步重逻辑，降低 FastAPI 事件循环阻塞风险。
- 增强快照与缓存并发安全（锁与事件协同）并补充回归测试。
